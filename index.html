Evolution

From beginner template to production-ready frontend.

This page began as an Elzero Web School exercise — a static agency landing page built with HTML and CSS. The visual design stayed intact; the engineering underneath was rebuilt for accessibility, maintainability, and real-world deployment.

View Source Code GitHub Profile

Before — Original Version

  • Basic HTML structure with unclosed tags and invalid nesting
  • No accessibility — missing ARIA, skip link, and keyboard support
  • Zero JavaScript — hover-only navigation, no touch or keyboard fallback
  • Broken and placeholder links (malformed WhatsApp URL, empty hrefs)
  • Monolithic CSS file with duplicated rules and display:none menu toggling
  • No SEO — missing meta description, Open Graph, and favicon

After — Upgraded Version

  • Semantic HTML — proper landmarks, single H1, corrected document structure
  • Full accessibility — ARIA labels, skip link, focus states, keyboard nav
  • SEO optimized — meta description, Open Graph tags, favicon, theme-color
  • Refactored CSS — design tokens, utilities, modular sections, focus rings
  • Responsive polish — dedicated tablet breakpoints and mobile layout fixes
  • Lightweight JS — nav toggle, scroll reveal, progressive enhancement

Thinking

Engineering decisions behind the upgrade.

Preserve, Don't Redesign

The original Leon layout is the design contract. Recruiters and users should see the same visual identity — proof that engineering skill and design skill are separate disciplines. Refactoring without redesign shows restraint and respect for existing systems.

Minimal JavaScript

No frameworks, no build step. A ~70-line vanilla script handles nav toggle and scroll reveal with progressive enhancement — content stays visible if JS fails. The right tool for a static portfolio piece is the smallest tool that works.

Refactor, Don't Rewrite

The CSS was reorganized into tokens, utilities, and section blocks — not replaced. This mirrors real-world legacy work where you improve maintainability incrementally instead of throwing away working code.

Maintainability First

CSS custom properties centralize colors, spacing, and transitions. One change to --main-color updates the entire system. Simple beats clever when another developer — or future you — needs to read this codebase.

Accessibility by Default

Skip links, ARIA attributes, keyboard menu control, and visible focus rings were added without altering the visual design. Accessibility is an engineering requirement, not a cosmetic layer applied at the end.

Performance Awareness

Lazy-loaded images, deferred scripts, reduced-motion media queries, and opacity/transform transitions instead of layout-thrashing display toggles. Small choices that compound in production.

Learned

What this project taught me about frontend engineering.

From Templates to Systems

Building a template teaches syntax. Upgrading one teaches judgment — knowing which problems matter (broken HTML, missing alt text, no focus states) and which don't (adding React to a static page). This project marks the shift from "does it look right?" to "is it correct, accessible, and maintainable?"

Legacy Code Is Real Work

Most professional frontend involves reading someone else's code — including your own from six months ago. Refactoring the Leon template simulated that experience: fixing unclosed tags, replacing hover-only patterns, and improving architecture without breaking what already worked visually.

Accessibility Without Compromise

Before this upgrade, the menu was invisible to keyboard users and screen readers had nothing to work with. Adding ARIA, semantic landmarks, and focus management improved the experience for everyone — and the page looks exactly the same. That's the goal.

Engineer, Not Just Implementer

A UI implementer copies a design pixel by pixel. A frontend engineer asks why the WhatsApp link is broken, whether the menu works on touch devices, and if the CSS will scale when the project grows. This case study is evidence of that mindset — documented, deliberate, and ready to discuss in an interview.
